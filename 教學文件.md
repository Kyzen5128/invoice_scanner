## 1. 系統架構說明

這個 App 使用「前端 + 後端 + 資料庫」三層架構：

```
┌─────────────────────────────────────────┐
│            Flutter App (前端)            │
│  掃描發票 → 儲存本地 → 同步後端 → 對獎  │
└────────────────┬────────────────────────┘
                 │ HTTP (REST API)
                 │ http://10.0.2.2/invoice_scanner
                 │ ↑ Android 模擬器連到電腦的特殊 IP
                 │
┌────────────────▼────────────────────────┐
│       PHP + Apache (後端)                │
│  XAMPP 管理，開啟 Apache 就自動啟動      │
│  C:\xampp\htdocs\invoice_scanner\*.php  │
└────────────────┬────────────────────────┘
                 │ PDO (PHP 內建)
                 │
┌────────────────▼────────────────────────┐
│       MySQL / MariaDB (資料庫)           │
│       由 XAMPP 在背景啟動管理            │
└─────────────────────────────────────────┘
```

### 為什麼用 PHP + Apache 而不是 Node.js？

| | PHP + Apache（本專案）| Node.js |
|---|---|---|
| **後端語言** | PHP | JavaScript |
| **伺服器** | Apache（XAMPP 管理）| Node.js 自己 |
| **啟動方式** | XAMPP 開 Apache → 自動好 | 要手動執行 `node server.js` |
| **適合場景** | 教學、快速開發 | 現代後端開發 |

PHP 後端放在 XAMPP 的 `htdocs` 資料夾，Apache 負責接收請求並轉給 PHP 處理，**XAMPP 開啟就全部好了**，不需要額外動作。

### 什麼是 URL？

URL 就是「網址」，告訴程式要去哪裡找資源。結構如下：

```
http://10.0.2.2/invoice_scanner/check.php
─┬──   ───┬───  ──────┬──────  ────┬────
 │         │           │             └ 路徑：要執行哪個檔案
 │         │           └ 目錄：htdocs 底下的哪個資料夾
 │         └ 主機：要連到哪台電腦
 └ 通訊協定：用 HTTP 傳輸
```

| 部分 | 這個專案的值 | 說明 |
|------|-------------|------|
| 通訊協定 | `http://` | 超文字傳輸協定 |
| 主機 | `10.0.2.2` | 模擬器內指向電腦的特殊 IP |
| 目錄 | `/invoice_scanner` | Apache htdocs 底下的資料夾名稱 |
| 檔案 | `/check.php` | 要執行的 PHP 檔案 |

### 怎麼找自己的 URL？

依照你的執行環境，主機的部分不同：

| 環境 | baseUrl |
|------|---------|
| Android 模擬器 | `http://10.0.2.2/invoice_scanner` |
| iOS 模擬器 | `http://localhost/invoice_scanner` |
| 實體手機（同一個 WiFi）| `http://你電腦的區網IP/invoice_scanner` |

**查詢你電腦的區網 IP（給實體手機用）：**

在 PowerShell 執行：
```powershell
ipconfig
```

找 `Wireless LAN adapter Wi-Fi` 底下的 `IPv4 Address`，例如：
```
IPv4 Address. . . . . . . . . . . : 192.168.1.105
```

那麼實體手機的 baseUrl 就要改成：
```dart
static const String baseUrl = 'http://192.168.1.105/invoice_scanner';
```

> **注意：** 電腦和手機必須連到同一個 WiFi，手機才連得到電腦的 Apache。

### 為什麼 Android 模擬器要用 `10.0.2.2`？

模擬器是一台「虛擬的手機」，有自己的網路環境：

- `localhost` = 模擬器自己（不是你的電腦）
- `10.0.2.2` = 你電腦的 localhost（由 Android 模擬器特別對應）

所以後端 URL 要寫 `http://10.0.2.2/invoice_scanner`。

### 資料流程

```
使用者掃描發票
    ↓
OCR 辨識文字 (Google ML Kit)
    ↓
解析欄位 (發票號碼、日期、金額、店家)
    ↓
進入明細頁讓使用者確認/修正
    ↓
點擊儲存
    ├─→ 寫入本地 JSON 檔（離線也能用）
    └─→ POST invoices.php → 同步到後端 MySQL

使用者點擊🏆對獎

    ↓

    ↓

使用者刪除發票
    ↓
刪除本地 JSON 檔內的紀錄
    ↓
DELETE invoices.php?number=xxx → 同步刪除後端
```

## 1.5 基本概念補充

### HTTP 方法（GET / POST / DELETE）

HTTP 是前後端溝通的語言，每一個請求都會帶一個「方法」告訴後端你想做什麼事：

| 方法 | 意思 | 日常比喻 | 這個專案的用途 |
|------|------|---------|--------------|
| **GET** | 取得資料 | 去圖書館「查」一本書 | 查詢發票、對獎 |
| **POST** | 送出資料（新增/更新）| 去圖書館「還」一本書（加入館藏）| 儲存發票 |
| **DELETE** | 刪除資料 | 去圖書館「報廢」一本書 | 刪除發票 |

```
Flutter App                          PHP 後端
────────────────────────────────────────────────
GET  /invoices.php        →  查詢所有發票，回傳 JSON
POST /invoices.php + JSON →  新增/更新一張發票
DEL  /invoices.php?number →  刪除這張發票
GET  /check.php           →  對獎，回傳中獎清單
```

**GET 和 POST 最大的差別：**
- GET：資料放在 URL 上，例如 `?period=11502`，網址看得到
- POST：資料放在請求的 body 裡，網址看不到，適合傳送完整的表單資料

---

### JSON — 前後端傳資料的格式

JSON（JavaScript Object Notation）是一種資料格式，前後端用它來傳遞資料。

**長這樣：**
```json
{
  "invoice_number": "WR-73786487",
  "period": "11502",
  "amount": 100,
  "invoice_date": "2026-02-22"
}
```

**為什麼用 JSON？**
- 純文字，任何程式語言都能讀懂
- 結構清楚，有 key 和 value
- 比 XML 簡潔很多

**這個專案的資料流：**
```
Flutter 送出：
  POST body = '{"invoice_number":"WR-73786487","period":"11502","amount":100}'
                ↓
PHP 接收：
  $body = json_decode(file_get_contents('php://input'), true);
  // $body['invoice_number'] = "WR-73786487"
                ↓
PHP 回傳：
  echo json_encode(['ok' => true, 'id' => 12]);
  // '{"ok":true,"id":12}'
                ↓
Flutter 解析：
  final data = jsonDecode(res.body);
  // data['id'] = 12
```

### CORS — 為什麼 PHP 要加 `Access-Control-Allow-Origin`？

**CORS（Cross-Origin Resource Sharing，跨來源資源共享）** 是瀏覽器的一個安全機制。

**問題背景：**

瀏覽器預設不允許一個網頁去讀取「不同來源」的資料。
例如你的 App 在 `10.0.2.2`，後端在 `10.0.2.2/invoice_scanner`，這就算「跨來源」。

```
Flutter App（http://10.0.2.2）
  ↓ 發請求
PHP 後端（http://10.0.2.2/invoice_scanner/invoices.php）
  ↓ 沒有 CORS header
瀏覽器/Flutter：❌ 拒絕！不同來源，不給讀
```

**解法：後端在回應加上這個 header，告訴對方「我允許你讀」**

```php
header('Access-Control-Allow-Origin: *');   // * 代表允許任何來源
```

加了之後：
```
Flutter App 發請求 → PHP 回應帶 CORS header → Flutter 成功讀到資料 ✅
```

**OPTIONS 預檢請求是什麼？**

瀏覽器在發 POST / DELETE 之前，會先偷偷送一個 OPTIONS 請求問後端：
「我等一下要送 POST，你允許嗎？」

所以 PHP 要先處理這個預檢，不然後端直接忽略，前端就會報錯：

```php
if ($_SERVER['REQUEST_METHOD'] === 'OPTIONS') {
    http_response_code(200);
    exit;  // 直接回應 OK，不做其他事
}
```

## 2. 環境安裝

### 2.1 安裝 XAMPP

XAMPP 是一個整合包，一次安裝好三個東西：
- **Apache**：網頁伺服器，負責接收 HTTP 請求並執行 PHP
- **MySQL/MariaDB**：資料庫，儲存發票和中獎號碼
- **PHP**：後端語言，處理 API 邏輯

1. 前往 `https://www.apachefriends.org` 下載 XAMPP
2. 執行安裝檔，一路按 Next
3. 安裝完成後開啟 **XAMPP Control Panel**
4. 點擊 **Apache** → **Start**，點擊 **MySQL** → **Start**
5. 兩個都變成綠色代表成功

### 2.2 建立 Junction（讓 XAMPP 找到你的 PHP 後端）

XAMPP 的 Apache 只認識 `C:\xampp\htdocs\` 底下的檔案。
我們的後端程式碼在 `D:\work\...\backend\`，用 **Junction（目錄捷徑）** 讓兩邊連起來。

在 PowerShell 執行一次（之後不需要再執行）：

```powershell
cmd /c mklink /J "C:\xampp\htdocs\invoice_scanner" "D:\work\flutter_lesson\invoice_scanner\backend"
```

建立成功後，`C:\xampp\htdocs\invoice_scanner\` 和 `D:\work\...\backend\` 是**同一份檔案**，
在 git 裡修改，Apache 也會看到最新版本。

### 3.4 期別（period）編碼說明

台灣統一發票每兩個月一期，期別編碼規則：

```
民國年份 + 該期的偶數月份（補零至兩位）

範例：
  2024年3月15日 → 民國113年 → 3-4月期 → "11304"
  2024年5月01日 → 民國113年 → 5-6月期 → "11306"
  2026年2月22日 → 民國115年 → 1-2月期 → "11502"
  2025年12月15日 → 民國114年 → 11-12月期 → "11412"
```

Flutter 中的計算函式（`lib/services/api_service.dart`）：

```dart
static String periodFromDate(DateTime date) {
  final roc = date.year - 1911;                              // 西元轉民國
  final evenMonth = date.month % 2 == 0 ? date.month        // 偶數月直接用
                                        : date.month + 1;   // 奇數月 +1
  return '$roc${evenMonth.toString().padLeft(2, '0')}';
}
```

## 4. 建立後端 API

後端由四個 PHP 檔案組成，全部放在 `backend/`：

```
backend/
├── db.php        ← 資料庫連線（被其他三個檔案引用）
├── invoices.php  ← 發票 CRUD（新增、查詢、刪除）
├── winning.php   ← 查詢中獎號碼
└── check.php     ← 對獎功能
```

### 4.1 db.php — 資料庫連線

```php
<?php
$pdo = new PDO("mysql:host=localhost;dbname=invoice_db;charset=utf8mb4", 'root', '');
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
$pdo->setAttribute(PDO::ATTR_DEFAULT_FETCH_MODE, PDO::FETCH_ASSOC);
```

- **PDO**：PHP 內建的資料庫連線介面，支援 prepared statement 防止 SQL injection
- 其他 PHP 檔案用 `require_once 'db.php'` 引用這個連線

### 4.2 invoices.php — 發票 CRUD

| HTTP 方法 | 動作 |
|-----------|------|
| GET | 查詢所有發票 |
| POST | 新增或更新一張發票（用 invoice_number + period 判斷是否已存在）|
| DELETE `?number=WR-xxx` | 依發票號碼刪除 |

**為什麼 DELETE 用 query string（`?number=xxx`）而不是 URL 路徑？**

PHP 每個 API 是一個獨立的 `.php` 檔案，無法像 Node.js/Express 那樣定義 `/invoices/number/:id` 這樣的路由。
用 `$_GET['number']` 讀取 query string 是 PHP 最直覺的做法。

### 4.3 check.php — 對獎

用 SQL JOIN 一次比對所有發票與中獎號碼：

```sql
SELECT i.invoice_number, w.prize_type, w.prize_amount
FROM invoices i
JOIN winning_numbers w
  ON i.period = w.period         -- 期別要一樣
 AND i.invoice_number = w.number -- 號碼要一樣
ORDER BY w.prize_amount DESC
```

只有 period 和 number 都相同，才算中獎。

### 4.4 REST API 說明表

| 方法 | URL | 用途 |
|------|-----|------|
| GET | `/invoice_scanner/invoices.php` | 查詢所有發票 |
| POST | `/invoice_scanner/invoices.php` | 新增或更新發票 |
| DELETE | `/invoice_scanner/invoices.php?number=xxx` | 依號碼刪除發票 |
| GET | `/invoice_scanner/winning.php` | 查詢所有中獎號碼 |
| GET | `/invoice_scanner/winning.php?period=xxx` | 查詢指定期別的中獎號碼 |
| GET | `/invoice_scanner/check.php` | 對獎 |

### 步驟一：新增 http 套件

Flutter 預設不包含發送 HTTP 請求的功能，需要安裝 `http` 套件。

在 `pubspec.yaml` 的 `dependencies` 區塊加入：

```yaml
dependencies:
  flutter:
    sdk: flutter
  http: ^1.2.0    # ← 加這行
```

然後在終端機執行：

```bash
flutter pub get
```

### 步驟二：新建 api_service.dart

在 `lib/services/` 資料夾新建 `api_service.dart`。
這個檔案集中管理**所有與後端的溝通**，其他地方只需要呼叫這裡的函式，不需要知道 URL 細節。

```dart
import 'dart:convert';
import 'package:flutter/foundation.dart';
import 'package:http/http.dart' as http;

class ApiService {
  // ① baseUrl：後端的根網址
  //    Android 模擬器用 10.0.2.2（等同於電腦的 localhost）
  //    Apache 預設 port 80，不需要寫 :80
  static const String baseUrl = 'http://10.0.2.2/invoice_scanner';

  // ② 將發票日期轉換為台灣統一發票期別代碼
  //    例：2026-02-22 → 民國115年1-2月 → "11502"
  static String periodFromDate(DateTime date) {
    final roc = date.year - 1911;                               // 西元轉民國年
    final evenMonth = date.month % 2 == 0 ? date.month         // 偶數月直接用
                                          : date.month + 1;    // 奇數月 +1 進位到下一個偶數月
    return '$roc${evenMonth.toString().padLeft(2, '0')}';       // 月份補零，確保兩位數
  }

  // ③ 新增或更新一張發票
  //    後端會用 invoice_number + period 判斷是否已存在
  //    存在 → 更新；不存在 → 新增
  static Future<int?> addInvoice({
    required String invoiceNumber,
    required String period,
    int amount = 0,
    String? invoiceDate,
    String? imagePath,
  }) async {
    // 組成要送給後端的 JSON 資料
    final body = {
      'invoice_number': invoiceNumber,
      'period': period,
      'amount': amount,
      'invoice_date': invoiceDate,
      'image_path': imagePath,
    };

    debugPrint('📤 [API] POST /invoices  送出: ${jsonEncode(body)}');

    final res = await http.post(
      Uri.parse('$baseUrl/invoices.php'),
      headers: {'Content-Type': 'application/json'},  // 告訴後端這是 JSON 格式
      body: jsonEncode(body),                          // 將 Map 轉成 JSON 字串
    );

    debugPrint('📥 [API] POST /invoices  收到 ${res.statusCode}: ${res.body}');

    if (res.statusCode == 200) {
      return jsonDecode(res.body)['id'] as int?;  // 回傳後端給的 MySQL id
    }
    throw Exception('addInvoice failed: ${res.statusCode} ${res.body}');
  }

  // ④ 依發票號碼刪除後端的發票紀錄
  //    為什麼用號碼而不是 id？
  //    因為 Flutter 本地用 UUID，後端用 MySQL 整數 id，格式不同無法對應
  //    發票號碼是兩邊都有的共同欄位
  static Future<void> deleteInvoiceByNumber(String invoiceNumber) async {
    // Uri.encodeComponent：將 "WR-73786487" 的 "-" 等特殊字元轉成 URL 安全格式
    final encoded = Uri.encodeComponent(invoiceNumber);

    debugPrint('📤 [API] DELETE /invoices.php?number=$encoded');

    final res = await http.delete(
      Uri.parse('$baseUrl/invoices.php?number=$encoded'),
    );

    debugPrint('📥 [API] DELETE /invoices.php  收到 ${res.statusCode}: ${res.body}');

    if (res.statusCode != 200) {
      throw Exception('deleteInvoiceByNumber failed: ${res.statusCode}');
    }
  }

  // ⑤ 對獎：請後端比對所有發票與中獎號碼
  //    回傳格式：{ count: 1, total_prize: 2000000, winners: [...] }
  static Future<Map<String, dynamic>> checkWinners() async {
    debugPrint('📤 [API] GET /check  發起對獎請求');

    final res = await http.get(Uri.parse('$baseUrl/check.php'));

    debugPrint('📥 [API] GET /check  收到 ${res.statusCode}: ${res.body}');

    if (res.statusCode == 200) {
      return Map<String, dynamic>.from(jsonDecode(res.body));
    }
    throw Exception('checkWinners failed: ${res.statusCode}');
  }
}
```

### 步驟三：修改 invoice_detail_provider.dart

**位置：** `lib/features/invoice_detail/presentation/invoice_detail_provider.dart`

這個檔案原本只負責把發票存到本地 JSON 檔。現在要在儲存完本地之後，**同步推送到後端資料庫**。

在檔案最上方加入 import：
```dart
import '../../../services/api_service.dart';  // ← 加這行
```

找到 `save()` 函式，在 `await _storage.saveInvoice(entityToSave);` 之後加入後端同步的程式碼：

```dart
Future<void> save() async {
  // 1. 確保照片從暫存路徑複製到 App 永久目錄
  final savedImagePath = await _storage.copyImageToAppDirectory(
    state.imageLocalPath, state.id
  );
  final entityToSave = state.copyWith(imageLocalPath: savedImagePath);

  // 2. 存到本地 JSON 檔（優先，確保離線也能用）
  await _storage.saveInvoice(entityToSave);

  // ↓↓↓ 新增這段：同步推送到後端 ↓↓↓
  try {
    final number = entityToSave.invoiceNumber;
    final date = entityToSave.date ?? entityToSave.scannedAt;

    // 只有發票號碼不為空才推送（空號碼無意義）
    if (number != null && number.isNotEmpty) {
      await ApiService.addInvoice(
        invoiceNumber: number,
        period: ApiService.periodFromDate(date),         // 從日期計算期別
        amount: (entityToSave.totalAmount ?? 0).round(), // double 轉 int
        invoiceDate: date.toIso8601String().substring(0, 10), // 只取 yyyy-MM-dd
        imagePath: entityToSave.imageLocalPath,
      );
    }
  } catch (e) {
    // 後端同步失敗不影響本地儲存，只記錄 log
    // 這樣離線狀態下 App 依然可以正常使用
    print('[InvoiceDetail] 後端同步失敗 (本地已儲存): $e');
  }
  // ↑↑↑ 新增結束 ↑↑↑
}
```

### 步驟四：修改 invoice_list_provider.dart

**位置：** `lib/features/invoice_list/presentation/invoice_list_provider.dart`

這個檔案原本的 `deleteInvoice()` 只刪本地 JSON。現在要**同時刪除後端的紀錄**，並且接收傳進來的發票號碼。

在檔案最上方加入 import：
```dart
import 'package:flutter/foundation.dart';        // ← 加這行（debugPrint 用）
import '../../../services/api_service.dart';     // ← 加這行
```

把原本的 `deleteInvoice(String id)` 改成以下版本，**新增 `invoiceNumber` 參數**：

```dart
/// 刪除特定發票（本地 + 後端同步）
///
/// [id] — Flutter 本地的 UUID，用來刪本地 JSON
/// [invoiceNumber] — 發票號碼，用來刪後端 MySQL（兩個 id 格式不同，各用各的）
Future<void> deleteInvoice(String id, {String? invoiceNumber}) async {
  try {
    // 1. 先刪本地 JSON（確保本地一定被清除）
    await _storage.deleteInvoice(id);

    // 2. 再同步刪除後端（失敗只記 log，不影響本地已完成的刪除）
    if (invoiceNumber != null && invoiceNumber.isNotEmpty) {
      try {
        await ApiService.deleteInvoiceByNumber(invoiceNumber);
      } catch (e) {
        debugPrint('[InvoiceList] 後端刪除失敗 (本地已刪除): $e');
      }
    }

    // 3. 重新載入清單，觸發 UI 更新
    await loadInvoices();
  } catch (e, st) {
    state = AsyncValue.error(e, st);
  }
}
```

**修改三：在 class 內新增 `_checkWinners` 方法**

在 `_confirmDelete` 方法旁邊加入以下完整方法：

```dart
/// 呼叫後端對獎，顯示中獎結果對話框
Future<void> _checkWinners(BuildContext context) async {
  // 先抓 navigator，避免 await 後 context 失效
  final navigator = Navigator.of(context, rootNavigator: true);

  // 顯示載入中轉圈
  showDialog(
    context: context,
    barrierDismissible: false,
    useRootNavigator: true,
    builder: (_) => const Center(child: CircularProgressIndicator()),
  );

  try {
    final result = await ApiService.checkWinners();

    // 等一個 frame，確保 dialog 動畫完成再 pop，避免 _debugLocked 錯誤
    await Future.delayed(Duration.zero);
    if (navigator.canPop()) navigator.pop();  // 關閉載入轉圈
    if (!context.mounted) return;

    // 解析回傳資料
    final int count = (result['count'] as int?) ?? 0;
    final int total = (result['total_prize'] as int?) ?? 0;
    final List winners = (result['winners'] as List?) ?? [];

    // 顯示對獎結果
    showDialog(
      context: context,
      builder: (c) => AlertDialog(
        shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(20)),
        title: Row(
          children: [
            Icon(
              count > 0 ? Icons.emoji_events_rounded : Icons.sentiment_neutral_rounded,
              color: count > 0 ? Colors.amber : Colors.grey,
            ),
            const SizedBox(width: 8),
            Text(count > 0 ? '恭喜中獎!' : '本次未中獎'),
          ],
        ),
        content: Column(
          mainAxisSize: MainAxisSize.min,
          crossAxisAlignment: CrossAxisAlignment.start,
          children: [
            Text('中獎張數：$count 張'),
            Text('總獎金：\$$total'),
            if (winners.isNotEmpty) ...[
              const Divider(height: 24),
              const Text('明細：', style: TextStyle(fontWeight: FontWeight.bold)),
              const SizedBox(height: 8),
              // 逐筆列出中獎發票
              ...winners.map((w) {
                final m = Map<String, dynamic>.from(w as Map);
                return Padding(
                  padding: const EdgeInsets.symmetric(vertical: 4),
                  child: Text(
                    '• ${m['invoice_number']}  ${m['prize_type']}  \$${m['prize_amount']}',
                  ),
                );
              }),
            ],
          ],
        ),
        actions: [
          TextButton(
            onPressed: () => Navigator.pop(c),
            child: const Text('關閉'),
          ),
        ],
      ),
    );
  } catch (e) {
    await Future.delayed(Duration.zero);
    if (navigator.canPop()) navigator.pop();
    if (!context.mounted) return;

    // 連線失敗時顯示錯誤訊息
    showDialog(
      context: context,
      builder: (c) => AlertDialog(
        title: const Text('對獎失敗'),
        content: Text('無法連線到後端伺服器：\n$e'),
        actions: [
          TextButton(
            onPressed: () => Navigator.pop(c),
            child: const Text('確定'),
          ),
        ],
      ),
    );
  }
}
```

## 6. 啟動順序與測試

### 6.1 每次開發前的啟動順序

**第一步：XAMPP 開啟 Apache + MySQL**
1. 開啟 XAMPP Control Panel
2. Apache → **Start**
3. MySQL → **Start**
4. 兩個都變成綠色就完成了

**第二步：Flutter run**

```bash
flutter run
```

就這樣，不需要其他動作。

### 6.2 確認後端是否正常（PowerShell 測試）

```powershell
# 查詢所有發票
Invoke-RestMethod -Uri "http://localhost/invoice_scanner/invoices.php"

# 手動新增一張測試發票
Invoke-RestMethod -Method Post `
  -Uri "http://localhost/invoice_scanner/invoices.php" `
  -ContentType "application/json" `
  -Body '{"invoice_number":"WR-73786487","period":"11502","amount":100,"invoice_date":"2026-02-22"}'

# 執行對獎
Invoke-RestMethod -Uri "http://localhost/invoice_scanner/check.php"
```

### 6.3 從 App 測試完整流程

1. 掃描發票（或手動輸入號碼與日期）
2. 點右上角勾號儲存
3. Flutter console 應出現：
   ```
   📤 [API] POST /invoices  送出: {"invoice_number":"...","period":"..."}
   📥 [API] POST /invoices  收到 200: {"ok":true,"id":1}
   ```
4. 點🏆對獎按鈕
5. Flutter console 應出現：
   ```
   📤 [API] GET /check  發起對獎請求
   📥 [API] GET /check  收到 200: {"count":1,"total_prize":2000000,...}
   ```
6. App 顯示中獎結果

## 7. 常見問題

### Q1：App 一直轉圈，後端沒有回應

確認順序：
1. XAMPP Control Panel → Apache 是否為綠色？
2. XAMPP Control Panel → MySQL 是否為綠色？
3. Flutter baseUrl 是否是 `http://10.0.2.2/invoice_scanner`？

### Q2：對獎回傳 count: 0

**最常見原因：期別（period）不符合。**

確認方法：
```powershell
# 查看發票的 period
& "C:\xampp\mysql\bin\mysql.exe" -u root -e "SELECT invoice_number, period FROM invoice_db.invoices;"

# 查看中獎號碼的 period
& "C:\xampp\mysql\bin\mysql.exe" -u root -e "SELECT period, number FROM invoice_db.winning_numbers;"
```

兩邊的 `period` 必須完全一樣才能對到。

### Q3：PHP 檔案沒有被 Apache 執行，直接顯示程式碼

原因：Junction 沒有建立，或 Apache 沒有啟動。

確認：
```powershell
# 確認 Junction 存在
Get-Item "C:\xampp\htdocs\invoice_scanner"

# 重新建立 Junction
cmd /c mklink /J "C:\xampp\htdocs\invoice_scanner" "D:\work\flutter_lesson\invoice_scanner\backend"
```

### Q4：XAMPP 和 PHP 的關係是什麼？

```
XAMPP 整合了三個東西：

Apache ──接收 HTTP 請求──→ 找到對應的 .php 檔 ──交給→ PHP 執行 ──回傳結果
MySQL ──提供資料庫服務──→ 被 PHP 用 PDO 連線操作

XAMPP 開啟 Apache + MySQL，整個後端就自動好了。
```
