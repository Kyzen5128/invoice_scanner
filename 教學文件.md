## 1. 系統架構說明

這個 App 使用「前端 + 後端 + 資料庫」三層架構：

```
┌─────────────────────────────────────────┐
│            Flutter App (前端)            │
│  掃描發票 → 儲存本地 → 同步後端 → 對獎  │
└────────────────┬────────────────────────┘
                 │ HTTP (REST API)
                 │ http://10.0.2.2:3000   ← Android 模擬器連到電腦的特殊 IP
                 │
┌────────────────▼────────────────────────┐
│         Node.js + Express (後端)         │
│           運行在你的電腦 port 3000       │
└────────────────┬────────────────────────┘
                 │ mysql2 (SQL 語法)
                 │
┌────────────────▼────────────────────────┐
│       MySQL / MariaDB (資料庫)           │
│       由 XAMPP 在背景啟動管理            │
└─────────────────────────────────────────┘
```

### 為什麼 Android 模擬器要用 `10.0.2.2` 而不是 `localhost`？

模擬器是一台「虛擬的手機」，它有自己的網路環境。在模擬器內：

- `localhost` = 模擬器自己（不是你的電腦）
- `10.0.2.2` = 你電腦的 localhost（由 Android 模擬器特別對應）

所以後端 URL 要寫 `http://10.0.2.2:3000`，不能寫 `http://localhost:3000`。

### 資料流程圖

```
使用者掃描發票
    ↓
OCR 辨識文字 (Google ML Kit)
    ↓
解析欄位 (發票號碼、日期、金額、店家)
    ↓
進入明細頁讓使用者確認/修正
    ↓
點擊儲存
    ├─→ 寫入本地 JSON 檔 (離線也能用)
    └─→ POST /invoices → 同步到後端 MySQL

使用者點擊🏆對獎
    ↓
GET /check
    ↓
後端 SQL JOIN 比對 invoices + winning_numbers
    ↓
回傳中獎清單 → App 顯示結果

使用者刪除發票
    ↓
刪除本地 JSON 檔內的紀錄
    ↓
DELETE /invoices/number/:invoiceNumber → 同步刪除後端
```

## 3. 建立資料庫

### 3.1 資料庫設計說明

我們需要兩張表：

**invoices（使用者的發票紀錄）**

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主鍵 |
| invoice_number | VARCHAR(20) | 發票號碼（如 WR-73786487）|
| period | VARCHAR(10) | 期別（如 11502 = 民國115年1-2月）|
| amount | INT | 消費金額 |
| invoice_date | DATE | 發票日期 |
| image_path | TEXT | 圖片路徑 |
| created_at | TIMESTAMP | 建立時間 |

**winning_numbers（中獎號碼，由管理員維護）**

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主鍵 |
| period | VARCHAR(10) | 期別 |
| prize_type | VARCHAR(20) | 獎別（特別獎、頭獎、六獎…）|
| number | VARCHAR(20) | 中獎號碼 |
| prize_amount | INT | 獎金金額 |

### 3.2 建立資料庫的 SQL（init.sql）

在 `backend/init.sql` 建立以下內容：

```sql
-- 建立資料庫
CREATE DATABASE IF NOT EXISTS invoice_db
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

USE invoice_db;

-- 建立發票表
CREATE TABLE IF NOT EXISTS invoices (
  id            INT AUTO_INCREMENT PRIMARY KEY,
  invoice_number VARCHAR(20)  NOT NULL,
  period        VARCHAR(10)  NOT NULL,
  amount        INT          NOT NULL DEFAULT 0,
  invoice_date  DATE,
  image_path    TEXT,
  created_at    TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uniq_invoice (invoice_number, period)   -- 同張發票不重複存入
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 建立中獎號碼表
CREATE TABLE IF NOT EXISTS winning_numbers (
  id           INT AUTO_INCREMENT PRIMARY KEY,
  period       VARCHAR(10)  NOT NULL,
  prize_type   VARCHAR(20)  NOT NULL,
  number       VARCHAR(20)  NOT NULL,
  prize_amount INT          NOT NULL DEFAULT 0,
  created_at   TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ⚠️ 注意：不要在這裡寫死中獎號碼
-- 中獎號碼的 period 必須和你掃描的發票 period 一致，才能對獎成功
-- 請依照「3.5 手動新增中獎號碼」的說明，在資料庫新增符合你測試發票期別的號碼
```

### 3.6 期別（period）編碼說明

台灣統一發票每兩個月一期，期別編碼規則：

```
民國年份 + 該期的偶數月份（補零至兩位）

範例：
  2024年3月15日 → 民國113年 → 3-4月期 → evenMonth=4 → "11304"
  2024年5月01日 → 民國113年 → 5-6月期 → evenMonth=6 → "11306"
  2026年2月22日 → 民國115年 → 1-2月期 → evenMonth=2 → "11502"
```

Flutter 中的計算函式：

```dart
static String periodFromDate(DateTime date) {
  final roc = date.year - 1911;                              // 西元轉民國
  final evenMonth = date.month % 2 == 0 ? date.month        // 偶數月直接用
                                        : date.month + 1;   // 奇數月+1
  return '$roc${evenMonth.toString().padLeft(2, '0')}';
}
```

### 4.2 後端架構（server.js）

後端使用 **Express** 框架，提供 REST API 給 Flutter 呼叫。

**完整的 `backend/server.js`：**

```javascript
const express = require('express');
const cors    = require('cors');
const mysql   = require('mysql2/promise');

const app = express();
app.use(cors());           // 允許跨來源請求（Flutter 需要）
app.use(express.json());   // 解析 JSON 格式的請求 body

// MySQL 連線池設定
const db = mysql.createPool({
  host: 'localhost',
  port: 3306,
  user: 'root',
  password: '',            // XAMPP 預設 root 無密碼
  database: 'invoice_db',
  waitForConnections: true,
  connectionLimit: 10,
});

// 健康檢查
app.get('/', (req, res) => {
  res.json({ ok: true, message: 'Invoice API is running' });
});

// ─── 新增/更新發票 ───────────────────────────────────────────────
// Flutter 每次儲存發票都會呼叫這個
app.post('/invoices', async (req, res) => {
  try {
    const { invoice_number, period, amount, invoice_date, image_path } = req.body;
    if (!invoice_number || !period) {
      return res.status(400).json({ error: 'invoice_number and period are required' });
    }

    // 先查是否已有同張發票（用 invoice_number + period 當唯一識別）
    const [existing] = await db.query(
      'SELECT id FROM invoices WHERE invoice_number = ? AND period = ?',
      [invoice_number, period]
    );

    let resultId;
    if (existing.length > 0) {
      // 已存在 → 更新金額、日期、圖片路徑
      await db.query(
        'UPDATE invoices SET amount=?, invoice_date=?, image_path=? WHERE invoice_number=? AND period=?',
        [amount || 0, invoice_date || null, image_path || null, invoice_number, period]
      );
      resultId = existing[0].id;
    } else {
      // 不存在 → 新增
      const [result] = await db.query(
        'INSERT INTO invoices (invoice_number, period, amount, invoice_date, image_path) VALUES (?,?,?,?,?)',
        [invoice_number, period, amount || 0, invoice_date || null, image_path || null]
      );
      resultId = result.insertId;
    }

    res.json({ ok: true, id: resultId });
  } catch (e) {
    res.status(500).json({ error: e.message });
  }
});

// ─── 查詢全部發票 ────────────────────────────────────────────────
app.get('/invoices', async (req, res) => {
  try {
    const [rows] = await db.query('SELECT * FROM invoices ORDER BY created_at DESC');
    res.json(rows);
  } catch (e) {
    res.status(500).json({ error: e.message });
  }
});

// ─── 刪除發票（依發票號碼）── Flutter 前端使用此路由 ────────────
// 注意：這個路由必須寫在 /invoices/:id 之前，否則 Express 會誤判
app.delete('/invoices/number/:invoiceNumber', async (req, res) => {
  try {
    const number = decodeURIComponent(req.params.invoiceNumber);
    await db.query('DELETE FROM invoices WHERE invoice_number = ?', [number]);
    res.json({ ok: true });
  } catch (e) {
    res.status(500).json({ error: e.message });
  }
});

// ─── 刪除發票（依 MySQL id）────────────────────────────────────
app.delete('/invoices/:id', async (req, res) => {
  try {
    await db.query('DELETE FROM invoices WHERE id = ?', [req.params.id]);
    res.json({ ok: true });
  } catch (e) {
    res.status(500).json({ error: e.message });
  }
});

// ─── 查詢某期中獎號碼 ────────────────────────────────────────────
app.get('/winning/:period', async (req, res) => {
  try {
    const [rows] = await db.query(
      'SELECT * FROM winning_numbers WHERE period = ?',
      [req.params.period]
    );
    res.json(rows);
  } catch (e) {
    res.status(500).json({ error: e.message });
  }
});

// ─── 對獎：比對使用者發票 vs 中獎號碼 ──────────────────────────
// 使用 SQL JOIN 一次比對所有發票
app.get('/check', async (req, res) => {
  try {
    const [rows] = await db.query(`
      SELECT i.id, i.invoice_number, i.period, w.prize_type, w.prize_amount
      FROM invoices i
      JOIN winning_numbers w
        ON i.period = w.period            -- 期別必須相同
       AND i.invoice_number = w.number    -- 號碼必須相同
      ORDER BY w.prize_amount DESC
    `);

    const total = rows.reduce((sum, row) => sum + row.prize_amount, 0);
    res.json({ count: rows.length, total_prize: total, winners: rows });
  } catch (e) {
    res.status(500).json({ error: e.message });
  }
});

// ─── 啟動伺服器 ──────────────────────────────────────────────────
app.listen(3000, () => {
  console.log('✅ Invoice API running at http://localhost:3000');
});
```

### 4.3 REST API 說明表

| 方法 | 路由 | 用途 |
|------|------|------|
| GET | `/` | 健康檢查，確認後端有在跑 |
| POST | `/invoices` | 新增或更新一張發票 |
| GET | `/invoices` | 取得所有發票 |
| DELETE | `/invoices/number/:invoiceNumber` | 依發票號碼刪除（Flutter 使用）|
| DELETE | `/invoices/:id` | 依 MySQL id 刪除 |
| GET | `/winning/:period` | 查某期的所有中獎號碼 |
| GET | `/check` | 對獎：回傳所有中獎發票與獎金 |

### 4.4 為什麼有兩個刪除路由？

Flutter 本地端用 UUID 當 ID，後端 MySQL 用整數 AUTO_INCREMENT 當 ID，兩者無法對應。
所以另外開了「依發票號碼刪除」的路由，讓 Flutter 用發票號碼（如 `WR-73786487`）來刪除。

**路由順序很重要！**
Express 從上到下匹配路由。`/invoices/number/:invoiceNumber` 必須寫在 `/invoices/:id` 之前，
否則 Express 會把 `"number"` 當成 `:id` 的值，找不到資料。

### 5.3 雙寫策略（Local + 後端）

每次使用者儲存發票，會同時寫入兩個地方：

```
儲存 → 本地 JSON 檔（優先，離線也能用）
     → 後端 MySQL（同步，失敗只記錄 log，不影響本地）
```

這樣設計的好處：
- 沒有網路時，App 還能正常使用
- 有網路時，資料自動備份到後端
- 後端資料可以用來做對獎比對

```dart
// invoice_detail_provider.dart 中的 save()
Future<void> save() async {
  // 1. 先存本地
  await _storage.saveInvoice(entityToSave);

  // 2. 再試著同步後端（失敗不影響使用）
  try {
    await ApiService.addInvoice(...);
  } catch (e) {
    print('[InvoiceDetail] 後端同步失敗 (本地已儲存): $e');
  }
}
```

### 5.4 刪除同步

刪除發票時，同樣同步刪除後端：

```dart
// invoice_list_provider.dart 中的 deleteInvoice()
Future<void> deleteInvoice(String id, {String? invoiceNumber}) async {
  // 1. 先刪本地
  await _storage.deleteInvoice(id);

  // 2. 同步刪除後端（失敗只記 log）
  if (invoiceNumber != null && invoiceNumber.isNotEmpty) {
    try {
      await ApiService.deleteInvoiceByNumber(invoiceNumber);
    } catch (e) {
      print('[InvoiceList] 後端刪除失敗 (本地已刪除): $e');
    }
  }
}
```

### 5.5 對獎功能

對獎按鈕呼叫後端 `/check`，後端用 SQL JOIN 做比對：

```sql
SELECT i.invoice_number, w.prize_type, w.prize_amount
FROM invoices i
JOIN winning_numbers w
  ON i.period = w.period            -- 期別要一樣
 AND i.invoice_number = w.number    -- 號碼要一樣
ORDER BY w.prize_amount DESC
```

**對獎能成功的條件：**
1. 發票的 `invoice_number` 和 `period` 都和中獎號碼一樣
2. 發票已經同步到後端資料庫
3. 中獎號碼已由管理員新增到 `winning_numbers` 表

## 7. 常見問題

### Q1：後端沒有回應，App 一直轉圈

**原因：** Node.js 後端沒有啟動。

**解法：** 確認後端 PowerShell 視窗還開著，看到 `✅ Invoice API running`。

### Q2：POST /invoices 回傳 500 Duplicate entry

**原因：** 同張發票（相同 invoice_number + period）已存在，且之前用了有問題的 `ON DUPLICATE KEY UPDATE`。

**解法：** 已改成顯式的 SELECT + UPDATE/INSERT 邏輯，不會再出現此問題。

### Q3：對獎回傳 count: 0，明明有掃同一張發票

**最常見原因：** 期別（period）不符合。

確認方法：
```powershell
# 查看發票的 period
& "C:\xampp\mysql\bin\mysql.exe" -u root -e "SELECT invoice_number, period FROM invoice_db.invoices;"

# 查看中獎號碼的 period
& "C:\xampp\mysql\bin\mysql.exe" -u root -e "SELECT period, number FROM invoice_db.winning_numbers;"
```

兩邊的 `period` 必須完全一樣，對獎才會成功。

### Q4：Android 模擬器連不到後端

確認順序：
1. XAMPP MySQL 是否啟動？
2. `node server.js` 是否在跑？
3. Flutter baseUrl 是否是 `http://10.0.2.2:3000`（不是 localhost）？

### Q5：XAMPP 和 Node.js 的關係是什麼？

| | XAMPP | Node.js |
|---|---|---|
| **功能** | 管理 MySQL 資料庫 | 執行你的 API 程式 |
| **啟動方式** | XAMPP Control Panel | 手動執行 `node server.js` |
| **互相依賴** | 無 | Node.js 需要 MySQL 已啟動才能連線 |

**XAMPP 不會幫你啟動 Node.js！** 這是最常搞混的地方。

## 附錄：資料庫常用指令

```powershell
# 進入 MySQL 互動介面
& "C:\xampp\mysql\bin\mysql.exe" -u root invoice_db

# 查看所有發票
SELECT * FROM invoices;

# 查看中獎號碼
SELECT * FROM winning_numbers;

# 新增中獎號碼
INSERT INTO winning_numbers (period, prize_type, number, prize_amount)
VALUES ('11504', '特別獎', 'AB-12345678', 2000000);

# 清空發票資料（測試用）
TRUNCATE TABLE invoices;

# 離開 MySQL 介面
exit;
```
